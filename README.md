# 🚗 Автомобильная приборная панель (Qt6 + ESP32 CAN Gateway)

Интерактивная цифровая приборная панель, разработанная на **C++ / Qt6 / QML**, работающая в связке с микроконтроллером **ESP32**, который выступает в роли универсального CAN-шлюза.

Проект поддерживает как автономную симуляцию физики автомобиля (без подключения к железу), так и чтение реальных данных из CAN-шины автомобиля через ESP32.

---

## 🧠 Архитектура проекта

Проект разделен на три независимых слоя, что позволяет изменять логику работы железа без необходимости переписывать интерфейс:

```text
┌─────────────────────────────────────────────────────────┐
│                       Qt / QML UI                       │
│            (Интерактивная приборная панель)             │
└────────────────────────────▲────────────────────────────┘
                             │ JSON (через Serial / Wi-Fi)
┌────────────────────────────┴────────────────────────────┐
│                   ESP32 Gateway Layer                   │
│         (Декодирование CAN + API сигналов)              │
└────────────────────────────▲────────────────────────────┘
                             │ RAW CAN данные
┌────────────────────────────┴────────────────────────────┐
│                    CAN Bus (500 kbps)                   │
│          (Бортовая сеть Renault / Lada / и др.)          │
└─────────────────────────────────────────────────────────┘
```

### Основной принцип:
Qt-приложение полностью изолировано от аппаратной логики. Оно лишь принимает стандартизированный JSON-пакет, парсит его и обновляет элементы интерфейса в реальном времени.

---

## ⚙️ Аппаратная часть: Режимы работы ESP32

Микроконтроллер ESP32 поддерживает 3 режима работы, переключение между которыми осуществляется через переменную `Mode mode = SIMULATION;` в коде прошивки:

1. **`SIMULATION`** — Автономный режим. ESP32 без подключения к машине генерирует физическую модель разгона, эмулирует переключение передач, формирует CAN-подобные кадры и отправляет их в Qt.
2. **`REAL_CAN`** — Режим работы в автомобиле. ESP32 слушает шину через встроенный контроллер TWAI (CAN), декодирует пакеты и транслирует их в UI.
3. **`LOG_ONLY`** — Режим черного ящика (Blackbox). Только логирование сырых CAN-кадров в Serial-порт для последующего анализа.

### 📡 Универсальный слой данных (Структура Signals)
Независимо от выбранного режима (симуляция или реальное авто), ESP32 приводит все данные к единой структуре:
```cpp
struct Signals {
  float rpm;        // Обороты двигателя
  float speed;      // Скорость автомобиля
  float coolant;    // Температура охлаждающей жидкости
  float fuel;       // Уровень топлива
  bool lights;      // Статус фар/габаритов
};
```

### 🔌 Поддерживаемые CAN ID (Пример для платформ Lada / Renault)

| Сигнал | CAN ID (HEX) | Статус |
| :--- | :--- | :--- |
| **RPM (Обороты)** | `0x186` | Поддерживается |
| **Speed (Скорость)** | `0x354` | Поддерживается |
| **Lights (Фары)** | `0x481` | Поддерживается |
| **Temperature (Температура)** | `0x551` | *В разработке* |
| **Gear (Передача)** | `0x421` | *В разработке* |

*Примечание: Реальные ID могут отличаться в зависимости от конкретной модели автомобиля (Logan, Sandero, Vesta).*

## 📦 Формат обмена данными (ESP32 → Qt)

Передача данных осуществляется по протоколу **Serial (через USB)** или по **Wi-Fi** (в будущем) в виде компактного JSON-пакета:

```json
{
  "rpm": 3200,
  "speed": 85,
  "gear": 3,
  "coolant": 78,
  "fuel": 62,
  "lights": 1
}
```

## 🛠 Требования к системе

### Для сборки Qt-приложения (Интерфейс):
* **Qt SDK** — Рекомендуется версия **Qt 6.x** (с компонентами *Qt Quick*, *QML* и *Qt5Compat*, если используются старые графические эффекты).
* **Инструменты сборки** — **CMake** (версии 3.16 и выше) и генератор сборки (**Ninja** или **Make**).
* **Компилятор** — MSVC (Windows), GCC (Linux), Clang (macOS).
* **IDE** — Qt Creator (рекомендуется), VS Code или CLion.

### Для прошивки ESP32 (Железо):
* Плата **ESP32 DevKit** (или любой аналог на базе ESP32).
* CAN-трансивер для согласования уровней (**SN65HVD230** рекомендуется для 3.3V логики ESP32, либо **TJA1050** с преобразователем уровней).
* Среда разработки **Arduino IDE 2.x** или **PlatformIO**.

## 🚀 Инструкция по запуску

### 1. Запуск приборной панели (Qt)

#### Вариант А. Через Qt Creator (Рекомендуемый)
1. Откройте **Qt Creator**.
2. Выберите `File` -> `Open File or Project...` и укажите путь к файлу **`CMakeLists.txt`** в корне репозитория.
3. Настройте комплект сборки (Configure Project).
4. Нажмите кнопку **Run** (`Ctrl + R` или `Cmd + R`).

#### Вариант Б. Сборка через терминал
```bash
# Перейдите в папку с проектом
cd path/to/dashboard

# Создайте папку для сборки и перейдите в нее
mkdir build && cd build

# Сгенерируйте файлы конфигурации CMake
cmake ..

# Скомпилируйте проект
cmake --build .

# Запустите исполняемый файл
# На Linux:
./dashboard
# На Windows:
dashboard.exe
```

### 2. Прошивка микроконтроллера (ESP32)
1. Перейдите в папку проекта `esp/esp_code1/` и откройте файл `esp_code1.ino` в **Arduino IDE**.
2. Установите в коде желаемый режим (например, `SIMULATION` для тестов на столе).
3. Выберите вашу плату ESP32 в меню инструментов.
4. Нажмите кнопку **Загрузка (Upload)**.
5. Для контроля данных откройте *Serial Monitor* на скорости **115200 baud**.
